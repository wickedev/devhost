Every directory gets its own localhost.

devhost gives each project — and each git worktree — its own loopback IP, so they can all bind :3000 at the same time. No port remapping, no dev servers killing each other, no code changes.

mechanism

How it works

devhost virtualizes the address, not the port — every port a project binds is covered at once, and URLs keep their familiar port numbers.

Marker

An empty .devhost file opts a tree in. Commit it; git worktrees inherit it.

Deterministic IP

The root path hashes into 127.77.0.0/16. Same path, same IP — no registry, no coordination.

Transparent rebinding

A PATH shim injects a tiny listen patch, so next dev or plain Node binds the project IP untouched.

Stable names

<dirname>.devhost resolves to the project IP — a browser URL that never changes.

localhost, restored

The daemon routes localhost:3000 by the caller's cwd — each worktree reaches its own server.

proof

Parallel E2E, for real

Browsers inherit the test runner's working directory — a Playwright Chromium in worktree A and one in worktree B both open localhost:3000 and each reaches its own server.

# ~/work/storefront                  # ~/work/storefront-cart-fix — at the same time
$ npx playwright test                 $ npx playwright test
webServer → 127.77.60.193:3000       webServer → 127.77.40.164:3000
chromium  → localhost:3000 ✓         chromium  → localhost:3000 ✓
